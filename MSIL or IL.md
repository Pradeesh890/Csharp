
MSIL or IL stands for Microsoft Intermediate Language or just Intermediate Language

This is CPU-Independent intermediate code
This is generated by all the .Net languages or .Net compilers

This is Language independent
This is stored inside .exe / .dll

MSIL uses a stack based instruction set
platform independent
Includes Instructions, [[Metadata]], Type info
this enable:
- JIT compilation
- security checks
- Optimization

Virtual Execution system (VES) is responsible for execution MSIL this handles
- Memory
- Type safety
- Exceptions 
this is implemented by CLR.
